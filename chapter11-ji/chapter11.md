# 第十一章 云

> 云在上面，人在下面。

后来我离开了 Uber。那几年 Uber 的自动驾驶项目很热，很多工程师相信车很快就可以自己开，人只需要坐在里面。后来出了事故，再后来，公司把自动驾驶业务卖给了 Aurora。很多人觉得这是一个商业决定，但事故改变了很多东西。我当时心里有一点不舒服——有些事情你以为会一直做下去，忽然就结束了。

那段时间我开始重新看机会。有几个公司给了我 offer：苹果、一家银行、还有 Oracle。Oracle 那时候其实并不是最热门的选择，很多工程师更愿意去新的创业公司，或者去 FAANG。那时候 Oracle 的股价大概在五十块左右，很多人觉得这是一家老公司，技术世界的注意力已经在别的地方。

我去面试ORACLE的时候，有一件事情让我印象很深。面试我的人，聊起来才发现是大学的师兄。我们本来不认识，但说着说着，发现都认识一些人，同一个系，还跟过同一位老师，就对上了。我们聊了很久，聊着聊着，忽然想起很多大学里的事、很多人、还有那时候的自己。后来 hiring manager 也和我聊了很久，很热情。不像面试，更像是在认真地介绍一个团队。那天面试结束以后，我忽然觉得，也许可以试一试。那天晚上我睡不着。

后来我去了 Oracle。很多朋友听说以后都笑我，觉得这是一个奇怪的选择——那时候硅谷有很多更「酷」的公司。但我还是去了。

后来我开始在 Oracle 做云上的 AI。OCI，Oracle Cloud Infrastructure。数据中心一排一排机器，服务器在机架里亮着灯，风扇一直在转，声音像远处的海。很多工程师每天写代码、写系统、写模型。云在天上——其实不是天上，是在世界各地的数据中心里，机器一排一排摆在那里，但工程师还是叫它「云」。

有一次我见到了 Larry Ellison。他已经七十多岁了，走路很慢，但说话很直接。很多工程师围在旁边听他说事情。技术世界也在变：AI 模型越来越大，代码越来越多，机器开始做以前只有人才能做的事情——写程序、写文档、回答问题。有时候我会想一个很简单的问题：如果机器也能写代码，人为什么还要写？如果机器也能思考，人为什么还要工作？在云上做 AI 的时候，这些问题不再只是哲学问题，它们变成工程问题：模型怎么训练、算力从哪里来、数据怎么流动、机器怎么学习。云在上面，人在下面，代码在中间。你写代码，机器也在写代码。

很多时候我会想起很久以前看到的一句话。那是在广州的一间办公室里，墙上挂着五个字：云层之上无风雨。很多年以后，我真的开始在「云」里工作。但云层之下，人还是要走路、要奔波、要做决定。机器在学习，系统在运行，服务器在很远的地方，风扇一直在转，数据在世界各地流动。有时候机器写得比人还快。

云在头顶。

人还在走路。
